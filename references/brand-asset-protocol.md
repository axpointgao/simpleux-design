# Brand Asset Protocol

本文件用于处理具体品牌、产品或明确客户项目中的品牌资产。正式客户交付、高保真批量制作、品牌化 deck、产品原型和对外发布材料必须执行完整协议；内部草稿、线框、结构讨论和早期低保真探索可以使用轻量品牌假设，但必须标注假设与后续替换风险。

## 核心顺序

资产 > 规范。先认出品牌，再谈风格。

当用户明确给出客户、品牌、产品或机构名称，且目标是正式客户交付或高保真产物时，这不是可选流程，而是开工门禁。即使用户没有主动提供素材，也必须先做基础品牌调研或一次性向用户索取素材；不得直接用通用金融、通用科技、通用医疗等视觉方向代替客户品牌。

当任务只是内部讨论、线框占位或结构梳理时，可以先用轻量品牌假设推进，但要清楚写明：

- 哪些 logo、色彩、字体、截图是待确认或临时占位。
- 哪些视觉判断不能视为客户品牌规范。
- 后续进入正式 deck、PDF、PPTX 或高保真原型前，需要回到完整品牌资产协议。

## 资产优先级

| 资产 | 优先级 | 是否必需 |
|---|---|---|
| Logo | 最高 | 必需 |
| 产品图 / 产品渲染图 | 很高 | 实体产品必需 |
| UI 截图 / 界面素材 | 很高 | 数字产品必需 |
| 色值 | 中 | 辅助 |
| 字体 | 低 | 辅助 |
| 气质关键词 | 低 | 辅助 |

## Step 1 · 一次问全

按这个清单问用户：

1. Logo
2. 产品图 / 官方渲染图
3. UI 截图 / 界面素材
4. 色值清单
5. 字体清单
6. Brand guidelines / Figma / 官网链接

如果用户已经给出明确品牌名但没有给资产，先并行做两件事：

1. 向用户一次性索取上述素材。
2. 搜官方渠道，建立临时品牌调研摘要。

不要一边缺 logo、缺 UI 截图、缺品牌色，一边直接进入高保真页面生产。

## Step 2 · 搜官方渠道

| 资产 | 搜索路径 |
|---|---|
| Logo | 官网 brand / press / press-kit / header inline SVG |
| 产品图 | 产品页 hero + gallery / 官方 launch film / 官方新闻稿 |
| UI 截图 | App Store / Google Play 截图 / 官网 screenshots / 官方视频截帧 |
| 色值 | 官网 CSS / Tailwind config / brand guide |
| 字体 | 官网字体引用 / Google Fonts / brand guide |

基础业务语境也要同步确认：

- 官网、公开介绍、业务范围和核心产品。
- 品牌主张、slogan、服务关键词和常用表达。
- 现有 App / 小程序 / 官网的视觉语言和信息架构。
- 与当前任务相关的业务模块、权益体系、会员体系、服务名称。
- 竞品或行业语境只作为理解背景，不得压过客户自身资产。

## Step 3 · 下载资产

### Logo

1. 独立 SVG/PNG。
2. 官网 HTML 里提取 inline SVG。
3. 官方社媒头像作为最后手段。

### 产品图

1. 官方 hero image。
2. 官方 press kit。
3. 官方 launch video 截帧。
4. Wikimedia Commons。
5. 实在没有时，才用 AI 生成兜底，但不要用 CSS/SVG 手画代替真实产品图。

### UI 截图

1. App Store / Google Play 截图。
2. 官网 screenshots。
3. 官方演示视频截帧。
4. 用户真实账号截图。

## Step 4 · 验证与提取

- Logo 要能打开，最好有深底 / 浅底两版。
- 产品图至少一张高分辨率主视角图。
- UI 截图要是最新版本，不能混入用户数据污染。
- 色值从真实资产里提取，不凭记忆猜。

### 质量门槛

遵守 `5-10-2-8`：

- 搜 5 轮
- 凑 10 个候选
- 选 2 个最好
- 每个至少 8/10

宁缺毋滥。

## Step 5 · 固化成 brand-spec.md

把这些写进去：

- 资产来源
- Logo 路径
- 产品图路径
- UI 截图路径
- 色板
- 字体
- 禁区
- 气质关键词
- 业务语境摘要
- 现有体验观察
- 本次设计可继承和需要强化的品牌线索

## 失败兜底

- Logo 找不到：停下问用户。
- 产品图找不到：优先 AI 参考生成，其次问用户，最后才诚实 placeholder。
- UI 截图找不到：向用户索取真实截图。
- 色值完全找不到：走设计方向顾问模式。

## 结构纪律

- HTML 必须引用 `brand-spec.md` 里的真实文件路径。
- Logo 和产品图都要用真实图片，不要手画。
- 想临时换色，先改 spec。
- 客户品牌资产决定主视觉；`assets/publisher/` 里的 SimpleUX 标识只作为设计方/出品方署名使用，不应覆盖、替代或干扰客户品牌系统。
