# Scenario Workflows：SimpleUX 方案场景入口

本文件是项目方案、POC 提案和设计提案的总入口。三类场景负责方案内容、咨询判断、材料整合、叙事结构、逐页稿和逐页表达规格；进入制作阶段后统一读取 `references/slide-decks.md` 执行 HTML-first deck 制作、视觉确认、署名封底、导出和 QA。

## 场景分流

### 项目方案

触发表达：项目方案、项目建议、合作方案、策略汇报、体验升级建议、项目启动方案。

使用目标：帮助客户理解项目如何开展，而不是说服客户为什么要做。重点是项目理解、体验问题界定、设计判断、策略建议、工作路径、交付与协作。

读取：`references/project-solution-workflow.md`。

### POC 提案

触发表达：POC、招标命题、投标设计方案、评分标准、设计响应、指定样稿命题。

使用目标：围绕单一设计命题组织 POC 设计方案表达，不覆盖完整标书章节，不写实施计划、团队能力、风控质保或商务响应。

读取：`references/poc-proposal-workflow.md`。

### 设计提案

触发表达：设计提案、多方向方案、方向评审、几套视觉方向、几套交互方向、早期设计定义。

使用目标：基于用户提供或确认的多个可比较方向，组织交互、视觉或综合方向表达，支持评审与后续详细设计衔接。

读取：`references/design-proposal-workflow.md`。

## 统一上游流程

三类场景都先执行：

```text
场景确认
→ 输入检查
→ 资料缺口/研究指引
→ 关键材料问询
→ 材料整合
→ 内容方向确认
→ 章节规划确认
→ 逐页稿确认
→ 逐页表达规格确认
→ slide-decks.md 制作
→ 场景化 QA
```

门槛、问询、逐页表达规格、业务价值连接和 QA 详见 `references/scenario-gates-and-qa.md`。

## 输入门槛

### 项目方案

- 客户/项目背景
- 项目目标
- 当前产品现状、已有诊断或截图
- 竞品或参考对象
- 项目范围或期望输出
- 业务目标、客户关注点或项目成效判断

### POC 提案

- 招标/POC 题目
- 设计要求和评分标准
- 客户行业语境
- 已有设计方向或方向判断
- 样稿状态：已有正式样稿 / 未提供正式样稿
- 输出格式和页数范围

### 设计提案

- 项目目标、调研洞察和设计策略
- 两个以上可比较方向，或需要基于资料生成方向建议草案
- 样稿状态：已有正式样稿 / 未提供正式样稿
- 交互层输入、视觉层输入和决策对象

## 输出边界

- 可以输出资料清单、研究 brief、材料整合、章节规划、逐页稿、逐页表达规格和 HTML-first deck。
- 不替代 deep research；只能给用户外部研究问题和输入模板。
- 不独立决定最终设计方向；方向建议草案必须由用户确认。
- 不生产正式交互/视觉样稿；未提供正式样稿时只能使用线框示意、结构占位或待替换样稿区。
- 不覆盖标书中的实施计划、团队能力、风控质保和商务响应等非设计方案章节。

## 与 HTML Deck 的关系

- 三类场景是内容与咨询表达层，不是新的 HTML 渲染层。
- 三类场景的制作阶段一律读取 `references/slide-decks.md`。
- `references/deck-quick-guide.md` 只服务普通短 deck、内部草稿和快速演示。
- `references/slide-decks.md` 消费已确认的逐页表达规格，负责多文件架构、2 页视觉确认稿、品牌资产、署名、封底、导出和 QA。
